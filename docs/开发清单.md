# Stable Placement Node Lease v2 开发清单

本文档记录当前 v2 能力。旧版 Grain Lease、Grain heartbeat、`Expire`、`ExpireDue`、`ExpireHeartbeats`、`LeaseVersion` 和 `PlacementStatusExpired` 清单已经被 Node Lease v2 取代，只能作为 v1 历史背景，不能用于判断当前 API 或部署能力。

设计目标：Lease 只属于 NodeSession；Placement 保存 Owner session 快照，但不拥有 TTL。

## 1. 公共模型

- [x] `Node` 持有 `NodeLease{Version, TTLMillis, ExpireAtUnixMilli}`。
- [x] `Placement` 持有 `OwnerNodeSessionID`，只保留 `Active` 和 `Released` 状态。
- [x] `PlacementRoute` 持有 `NodeLeaseVersion` 和进程内 `ValidUntil`。
- [x] 删除 Grain 级 Lease、到期状态和 `Expire` API。
- [x] `Lookup` 返回可路由快照；Allocate、Renew、Transfer、Recover 返回的 Placement 不是路由授权。

## 2. Node Lease 配置与时间

- [x] `NodeLeaseConfig` 在构造 Memory/Redis 实例时传入，实例创建后不可修改。
- [x] `DefaultNodeLeaseConfig()` 返回一分钟 TTL。
- [x] 构造函数对 `TTL <= 0` 返回 `ErrInvalidNodeLeaseTTL`。
- [x] RegisterNode 和 ReplaceNodeSession 把 TTL 毫秒值持久化到 `Node.Lease.TTLMillis`。
- [x] RenewNode 使用 Node 中持久化的 `TTLMillis`，不使用续约进程自身的构造配置。
- [x] Redis 注册、续约、Lookup、分配、迁移、恢复和过期判断使用 Lua 内的 Redis `TIME`。
- [x] Memory 使用同一个内部 clock 快照完成判断。
- [x] RenewNode 在 Lease 到期后返回 `ErrNodeLeaseExpired`，不能复活旧 session。

## 3. Node 生命周期

- [x] RegisterNode 校验 `NodeIdentity == NodeType/NodeGroup/NodeName`。
- [x] 首次 RegisterNode 强制进入 Active，Lease 版本从 1 开始。
- [x] 同 session 重复 RegisterNode 是幂等成功，不续约、不重复发布事件。
- [x] 不同 session 调用 RegisterNode 返回 `ErrInvalidNodeSession`，必须显式 ReplaceNodeSession。
- [x] Offline 或已过期的同 session 不能通过 RegisterNode 复活。
- [x] ReplaceNodeSession 创建新的独立 Lease，并发布 NodeReplaced。
- [x] 新 session 不继承旧 session 的 Placement。
- [x] ExpireNodeLeases 只推进 Node 为 Offline，并保留 Node tombstone 和 Placement 索引。

## 4. Placement 生命周期

- [x] Node Lease 到期会立即使关联 Grain 逻辑不可路由，不逐条改写 Placement。
- [x] 逻辑失效的 Placement 仍保持 Active 并保留 Owner 快照，供 FindByNode、Recover 和 Transfer 使用。
- [x] Lookup 对逻辑失效记录返回 `ErrPlacementNotFound`，Exists 返回 false。
- [x] Allocate 遇到仍保留但 Owner 不可用的 Active Placement 返回 `ErrPlacementOwnerUnavailable`，不自动换 Owner。
- [x] Directory.Renew 只校验 Placement、Owner session、版本和 Node Lease，并写审计；不延长任何 TTL。
- [x] Release 是显式终止操作；Owner session 未被替换时，即使 Node Offline 或 Lease 到期也允许释放。
- [x] Transfer 是显式管理迁移，可迁移健康或不可用 Owner 的 Placement。
- [x] Recover 仅用于 Owner 缺失、Offline、Lease 到期或 session 不匹配；健康 Owner 必须使用 Transfer。
- [x] Owner 变化只由 Transfer、Recover、Release 或 Released 后重新 Allocate 发生。

## 5. 缓存与事件

- [x] Lookup 生成的 `ValidUntil` 是进程内保守截止时间，不序列化到 Redis。
- [x] Redis Lookup 用同一次 Redis `TIME` 返回剩余 TTL；Go 从请求开始时刻计算 `ValidUntil`，网络耗时只缩短本地窗口。
- [x] 缓存命中必须满足当前时间早于 `ValidUntil`。
- [x] NodeLeaseExpired、NodeReplaced、NodeDraining、NodeMarkedInvalid 和 NodeUnregistered 按 NodeIdentity 清缓存。
- [x] 未知或字段不完整事件保守清空全部缓存。
- [x] PlacementRenewed 是审计事件，不作为缓存失效事件。

## 6. Redis v2 隔离

- [x] Redis 数据、Stream、Pub/Sub、consumer group 全部使用 v2 namespace。
- [x] v2 不读取、修改或删除 v1 key。
- [x] v1 key 预置时不影响 v2 行为。
- [x] 本库只暴露 `redis.NamespaceVersion` 和 `redis.NamespacePrefix` 供部署检查，不探测其他进程或 workload。
- [x] v1 key 不自动清理，稳定后由运维人工处理。

## 7. Memory / Redis 一致性

- [x] 两种实现对 TTL 配置、NodeIdentity 校验、session、到期边界和领域错误保持一致。
- [x] 两种实现都保留逻辑失效 Placement，且只允许显式 Recover/Transfer 改 Owner。
- [x] Memory 使用锁内 clock 快照；Redis 使用 Lua 和 Redis `TIME` 提供原子判断。
- [x] FindByNode 负 cursor 返回错误，越过当前索引的 cursor 返回空页。

## 8. 部署门禁

- [ ] 所有 v1 writer、Node、scanner 和 consumer 已停止，否则不得开放 v2 业务流量。
- [ ] 旧 Grain 执行已停止，所有进程内 Placement 缓存已清空。
- [ ] 部署检查确认 `redis.NamespaceVersion == "v2"` 且 `redis.NamespacePrefix == "sp:{stable-placement}:v2:"`。
- [ ] 预置 v1 key 的验收证明 v2 不读写 v1。
- [ ] 全部 v2 writer、Node、Node Lease scanner 和 Stream consumer 一次性启动并完成健康检查后才开放流量。
- [ ] 已确认首笔 v2 业务写入后禁止直接回滚 v1。
- [ ] 按 [Node Lease v2 冷切换手册](./node-lease-v2-cutover.md) 留存部署证据。

## 9. 历史清单

以下 v1 能力已废止，不是待恢复功能：

- Grain 级 Lease、LeaseVersion、LeaseTTL 和 ExtendTTL。
- Grain 级 heartbeat、Expire、ExpireDue 和 ExpireHeartbeats。
- Placement 的 Expired 状态和逐 Grain 到期改写。
- RegisterNode 覆盖不同 session，或通过续约/注册复活过期 session。
